HOM

HOM — Harness Orchestration Management

A Rust TUI terminal multiplexer that spawns 7 AI coding agent CLIs in visual panes. Each pane runs a real terminal emulator (vt100 default, libghostty-rs target), mapped cell-by-cell into ratatui buffers. The orchestrator coordinates inputs, outputs, and YAML-driven DAG workflows between harnesses.

7-Crate Workspace

Strict dependency rules: hom-core is the root with zero internal deps. Each layer depends only on hom-core. Only hom-tui and main.rs pull everything together.

Data Flow — The TUI-inside-TUI Pipeline

Every byte from a harness flows through this pipeline. A bug anywhere means garbled output.

Tier 1 has full orchestration with structured sideband channels. Tier 2 is headless/limited steering via screen parsing only.

Workflow Engine — YAML DAG Execution

Workflows define steps with dependencies. The engine builds a DAG (petgraph), does topological sort, and executes steps across harnesses with templating, retries, conditions, and SQLite checkpointing.
